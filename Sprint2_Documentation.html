تقرير Sprint 2 - Coffee Shop Management System

هذا ملف HTML واحد فقط يحتوي شرح المشروع ومتطلبات Sprint 2 بطريقة مبسطة. أسماء الكلاسات والدوال مكتوبة كما هي في الكود، أما الشرح فهو باللغة العربية لتسهيل المذاكرة والمناقشة.

مهم: هذا الملف خاص للمذاكرة والشرح. ملف Javadoc الرسمي في Maven ينتج ملفات HTML كثيرة بشكل طبيعي، لكن هذا الملف هو نسخة مبسطة في صفحة واحدة فقط.

2. فكرة النظام

النظام يدير عمليات مقهى بشكل عملي: تسجيل الدخول، إنشاء حسابات، استرجاع كلمة المرور، تغيير كلمة المرور وسؤال الأمان، إدارة التصنيفات والمنتجات، إنشاء الطلبات، حفظ الفواتير، عرض تفاصيل الطلبات، إدارة المستخدمين، وإصدار تقارير مبيعات حقيقية من قاعدة البيانات.

3. متطلبات Sprint 2 وكيف تحققت

المتطلبكيف تحقق في المشروع
Version Control تم استخدام Git ورفع المشروع إلى GitHub.
At least 7 classes المشروع يحتوي أكثر من 60 ملف Java.
Encapsulation الحقول في الموديلات private وتوجد constructors و getters/setters.
Inheritance User extends Person، Manager extends User، Employee extends User، Customer extends Person، VIPCustomer extends Customer.
Polymorphism التعامل مع Manager و Employee كـ User، واستخدام Payable payable = order.
Abstract Class Person يحتوي الحقول المشتركة والدالة abstract getRoleDescription().
Interfaces Authenticatable، Manageable، Repository، Payable، Printable، Discountable.
Method Overriding مثل getRoleDescription() و print() و calculateTotal() و calculateDiscount().
Exception Handling استخدام custom exceptions و try/catch في الواجهات والخدمات والمستودعات.
Javadoc التوثيق موجود داخل ملفات Java ويمكن توليده بالأمر mvn javadoc:javadoc.
Database Schema قاعدة MySQL باسم coffee_shop_db وجداول users, categories, products, customers, orders, order_items.
GUI Foundation واجهات Swing: Login, Main, Products, Categories, Orders, Invoices, Reports, User Management, Account Settings.
Database Connection Integration الاتصال يتم عبر DatabaseConfig و DatabaseManager و repositories.
Running System المشروع يترجم ويعمل باستخدام Maven.

11. مفاهيم OOP بالتفصيل

Encapsulation

كل الحقول المهمة داخل الموديلات private، ويتم الوصول لها من خلال getters و setters. 

private String name;
public String getName()
public void setName(String name)

Abstraction

موجود في Person لأنه abstract class يحتوي بيانات مشتركة ودالة abstract. 

public abstract class Person
public abstract String getRoleDescription();

Inheritance

User extends Person
Manager extends User
Employee extends User
Customer extends Person
VIPCustomer extends Customer

Interfaces and Implements

AuthService implements Authenticatable
CategoryService implements Manageable<Category>
ProductService implements Manageable<Product>
Order implements Payable, Printable
Invoice implements Printable
VIPCustomer implements Discountable

Polymorphism

يظهر عندما يتعامل الكود مع subclass من خلال superclass أو interface.

18. الإجابة المختصرة للمناقشة

المشروع هو نظام إدارة مقهى بلغة Java. استخدمت Maven لتنظيم المشروع، وSwing لبناء الواجهة، وMySQL لتخزين البيانات، وJDBC للاتصال. قسمت المشروع إلى models و services و repositories و gui و database و exceptions و interfaces. طبقت مفاهيم OOP مثل encapsulation و inheritance و polymorphism و abstract class و interfaces و overriding و overloading و inner class و exception handling. النظام يعمل فعليا، يفتح واجهة رسومية، يتصل بقاعدة البيانات، ويدير المستخدمين والمنتجات والطلبات والفواتير والتقارير.
